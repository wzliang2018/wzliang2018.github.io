Hexo文件迁移

发表于 2018-08-12 | 分类于 Hexo使用

这几天在重装电脑系统，将用不惯的Win10换成了习惯的Win7，所以Hexo就得重新捣鼓。
整个过程挺简单的，基本步骤如下：

1.将themes, source, _config.yml 这3个文件拷贝出来放好；

2.按照可能是最详细的 Hexo + GitHub Pages 搭建博客的教程 这篇文档重新搭建Node.js和Git的环境，Github配置，安装配置Hexo，关联关联 Hexo 与 GitHub Pages就OK了;
3.把前面拷贝好的文件拷贝到文件路径进行替换。

如果是同一台电脑更换不同的文件路径，那就更简单了，在将要放置博客文件的路径下点击右键，选择Git Bash Here，然后将原来路径下博客文件的==全部 #e91e63==拷贝到该路径下，完成。

博客初记

发表于 2018-08-11 | 分类于 Hexo使用

这个博客花了两天时间，算是初步建立了，后面还要完善不少功能，刚进坑，所以只能慢慢摸索了。好在用Hexo建立博客的网友实在是不少，很多人都有很好的总结，因此倒是不怕做不好，后面熟悉且页面定型后，最关键的应该就是是否有耐心记录自己。

开始的纠结

在打算建立博客后，自己经历了一系列的内心纠结。

  1. 是否有必要自建博客。现在有很多平台可以提供自己写博客（例如简书和cnblog似乎就不错，特别是简书），自己要不要花费时间建一个？感觉自己时间都不够用。最后觉得自己还能够挑战一下自己，而且平台的博客有广告，自己建立博客相对自由一些。
  2. 是否要买域名和服务器。自建博客的方法网上提供了不少，有那么多现成好用的自媒体平台，你为什么还需要老掉牙的独立博客？就提到好多种组合，考虑的因素有方式有技术、是否要买服务器、是否要买域名、费用，其中让我最纠结的就是要不要买域名了，前天晚上下班一回到家就在万网一边查可用的域名，一边想怎么想个好的域名，而且还要.com的域名，结果就是想了一晚上，直到睡觉也没有想到并找到一个可用的域名，郁闷的睡觉了。睡醒后觉得其实自己并没有必要非要一个域名，现在自己根本就是一个没有知名度也没有太多写作基础的博客入门者，根本没必要非要掏钱买个域名或者搞个服务器，先写着，如果自己觉得可以被外界所熟悉了，再考虑购买域名并推广自己的博客。不再纠结购买域名和服务器之后，从下面的表格就可很好的选出来了，用Hexo + GithubPage的方式最简单，不用交钱，入门门槛最低。

  1. 哪种博客的主题符合自己审美。博客的主题关乎自己审美的问题，所以这个问题很严重也很核心，因此花了不少时间来纠结。Hexo的Theme有200多款，但是一个个看下来，符合自己审美的也就2个，maupassant和Next，其中maupassant是最喜欢的，因为真的很简洁，我非常喜欢这种极简风格，但是maupassant提供的方法有点问题，我不想再把时间拖下去了，因此选择了文档非常详细，风格也很简洁的Next。现在看来文档详细的Next真的很用心，很适合我这种不想花太多心思再倒腾的人。

因为不懂，搭建博客一开始参考了很多博客，但我发现这篇 可能是最详细的 Hexo + GitHub Pages 搭建博客的教程最合适用Hexo + githubPage搭建博客，步骤详细，非常适合我这种小白。如果一开始就能找到这篇文章，或许我都不用太多时间来搭建博客。

关于Markdown编辑器

发表于 2018-08-11 | 分类于 Hexo使用

一开始用的的是马克飞象的Markdown编辑器，没找到其导出.md的位置，通过复制到.md文件的方式，发现表格和标题都有问题，折腾了好久才修改完。其实马克飞象有导出.md文件的功能，但是最终在博客上预览时表格和标题都有问题，不能正确显示。后面看到好用的Markdown编辑器一览列举了很多编辑器，一个一个试用，才发现小书匠是最好用的。

  1. dillinger的界面比较简洁，但是一直无法正常输入中文字体，很不好用。
  2. Cmd Markdown 对博客的文章标题不友好，而且奇葩的是还没有添加文章标题的机会！
  3. 其他，例如MarkdownPad、Atom、VSCODE等都要下载安装软件，太麻烦了，特别是VSCODE。
  4. 最后试了小书匠，好用！简直就是为了写博客存在的，非常方便，文章标题、标签等在导出后均没有异常，一步到位，实在是好用。
